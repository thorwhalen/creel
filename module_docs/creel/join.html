creel.join

Join the grammar and the extraction-metadata layers into a runnable plan.

Per decision D7 this is a pure equijoin by element id: for every node-type and edge-type in the GraphSpec, find its ExtractorBinding (if any) and resolve the concrete Extractor. Elements with no binding fall back to a schema_as_extractor factory when one is supplied (the progressive-disclosure default), otherwise they are reported as unbound.

The result, a ResolvedPlan, is what the facade maps over.

class creel.join.ResolvedPlan(steps: tuple[ResolvedStep, ...], unbound: tuple[str, ...])[source]

The full set of resolved extraction steps plus any unbound element ids.

class creel.join.ResolvedStep(element_id: str, element_type: ElementType, extractor: Extractor, binding_source: str, element_types: tuple = ())[source]

One element of the plan: the element(s), the resolved extractor, and the source.

element_types is the cluster this step covers (D-OP8); for an ordinary per-element step it is just (element_type,).

creel.join.SchemaAsExtractor

A factory (element_type, spec) -> Extractor used when an element has no binding.

alias of Callable[[ElementType, GraphSpec], Extractor]

creel.join.join(spec: GraphSpec, bindings: ExtractorBindings | None, *, schema_as_extractor: Callable[[ElementType, GraphSpec], Extractor] | None = None) → ResolvedPlan[source]

Equijoin spec with bindings by element id into a ResolvedPlan.

Cluster bindings (binding.elements) are resolved first into one multi-element step each (invoked once). Remaining elements resolve per-element. Node-types are produced before edge-types so the facade adds nodes before the edges that reference them.
