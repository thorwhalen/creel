creel.graph

creel graph layer: the in-memory LPG and its canonical JSON serialisation.

Re-exports the graph model (creel.graph.model) and the canonical JSON codec (creel.graph.canonical).

class creel.graph.Edge(id: str, source: str, target: str, type: str, attributes: Mapping[str, ~typing.Any]=<factory>)[source]

An immutable view of a graph edge: a first-class, identity-bearing relation.

class creel.graph.Graph[source]

A labeled property (multi-)graph with stable ids on nodes and edges.

Edge ids are unique across the whole graph (not merely per node pair); this is what makes the canonical JSON edge-by-id map and per-element provenance possible (decision D4/D8).

creel.graph.from_canonical_dict(data: Mapping[str, Any], *, validate: bool = True) → Graph[source]

Rebuild a Graph from a canonical dict.

With validate=True (default) the document is checked against CANONICAL_GRAPH_SCHEMA before loading, so malformed input fails fast with a clear error rather than a confusing KeyError mid-build.

creel.graph.to_canonical_dict(graph: Graph, *, spec: GraphSpec | None = None) → dict[str, Any][source]

Build the canonical dict for graph (edges id-sorted; ready to serialise).

If spec is given, a lightweight {"id", "version"} reference is recorded so a reader knows which grammar the instance claims to conform to.

creel.graph.to_canonical_json(graph: Graph, *, spec: GraphSpec | None = None, indent: int = 2) → str[source]

Serialise graph to canonical JSON text — deterministic and git-diffable.

Keys are sorted and edge arrays are id-sorted, so the same graph always yields byte-identical output regardless of insertion order.
