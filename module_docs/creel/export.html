creel.export

Export adapters — interchange formats for the canonical graph (EPIC 3.4 / 8.4).

Deterministic, dependency-free emitters that let the extracted graph flow into the wider ecosystem without coupling the core to any DB/library (decision D2/D10):

  • to_jgf() — JSON Graph Format (the de-facto JSON interchange).

  • to_graphml() — GraphML (XML; Gephi/yEd/NetworkX).

  • to_cypher() — parameterized Cypher statements as data (never string interpolation — injection-safe; bind the params with a Neo4j driver).

  • to_turtle() — RDF-star Turtle; attributes-on-edges become annotations on quoted triples (<< s p o >>), the lossless LPG↔RDF-star mapping (report R01).

All outputs are id-sorted → stable and git-diffable.

creel.export.to_cypher(graph: Graph) → list[tuple[str, dict[str, Any]]][source]

Return (statement, params) pairs to load the graph into Neo4j safely.

Relationship types cannot be parameterized in Cypher, so a generic :REL is used with a type property. Bind params via the driver — never interpolate.

creel.export.to_graphml(graph: Graph) → str[source]

Emit GraphML XML (string attributes; one <key> per attribute, id-sorted).

creel.export.to_jgf(graph: Graph) → dict[str, Any][source]

Project to JSON Graph Format (v2-style: nodes as an id→object map).

creel.export.to_turtle(graph: Graph, *, base: str = 'https://creel.dev/ns/') → str[source]

Emit RDF-star Turtle; edge attributes annotate quoted triples << s p o >>.

This is the lossless LPG↔RDF-star export (report R01). Caveat: parallel edges of the same (source, type, target) annotate the same quoted triple — distinguish them downstream by also reifying on the edge id if needed.
