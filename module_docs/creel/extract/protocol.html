creel.extract.protocol

The extractor strategy contract: what an extractor is given, and what it returns.

Per decision D5 an Extractor is a callable strategy — any (ExtractionContext) -> Extraction — dispatched per grammar element. A plain function is a valid extractor; no inheritance required (composition over OOP). The three built-in families (pattern, query, LLM) all satisfy this one Protocol.

An extractor bound to a node-type produces ExtractedNode instances; one bound to an edge-type produces ExtractedEdge instances; a richer extractor (e.g. an LLM reading prose) may return both. Each extracted element may carry an Evidence record for auditability.

class creel.extract.protocol.ExtractionContext(element_id: str, element_type: ElementType, sources: SourceBundle, spec: GraphSpec, cache: Cache = <factory>, services: Mapping[str, ~typing.Any]=<factory>, config: Mapping[str, ~typing.Any]=<factory>, element_types: Sequence[ElementType] = ())[source]

Everything an extractor needs to do its job for one grammar element.

element_id is the taxonomy id being extracted (a node-type or edge-type id); element_type is its typed schema (whose attribute description``s drive the schema-as-extractor default). ``services carries injected dependencies (an LLM client, an entity resolver, …) so the core never imports a provider SDK.

element_types: Sequence[ElementType] = ()

The cluster of element types this pass covers (D-OP8). Defaults to just (element_type,); a cluster extractor reads all of them and emits instances of each in a single LLM pass. element_id/element_type are the primary (first) of the cluster, so single-element extractors are unaffected.
