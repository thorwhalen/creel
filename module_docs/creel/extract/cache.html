creel.extract.cache

Caching for expensive (e.g. LLM) extraction calls.

Per decision D11 caching is exact-match and deterministic — a stable key derived from everything that affects the result — never semantic/approximate (which would break reproducibility and auditability). The Cache Protocol lets a persistent backend be injected via the extraction context; the default is a no-op.
