creel.resolve

Entity resolution — merge nodes that name the same real entity (decision #14 / D-OP8).

Messy multi-source documents (field-written, non-native English, OCR’d) name the same entity in different ways — “Foundation Alpha” / “Alpha Fnd. (MFA)” / “the Alpha foundation”. Per the changed decision #14 (report R14 §5), resolution is a required, pluggable cascade — blocking → matching → merging — defaulting to cheap deterministic strategies (exact id → normalize-before-merge → registry) with hooks for embedding/LLM adjudication.

A Resolver decides whether two nodes are the same entity. resolve_graph() applies one as a post-extraction pass: it blocks by node type, clusters by transitive sameness (union-find), merges each cluster into one canonical node, and remaps every edge — non-destructively returning a new graph with a merges report.

creel.resolve.resolve_graph(graph: Graph, resolver: Resolver, *, context: Any = None) → Graph[source]

Return a new graph with same-entity nodes merged and edges remapped.

Blocking (by node type-tuple) → matching (resolver.same) → merging (union-find; one canonical node per cluster, attributes first-wins) → edge remap. The original graph is untouched; the result’s report["merges"] lists each merge.
