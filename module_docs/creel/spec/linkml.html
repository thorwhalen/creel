creel.spec.linkml

LinkML bridge + schema codegen for the grammar (EPIC 2.4, decision D3).

LinkML is creel’s optional build-time authoring front-end, not the runtime SSOT (the Python GraphSpec is). This module bridges the two and generates validation artifacts — all dependency-free for the common subset (a LinkML schema is YAML with classes / attributes / enums), so it is fully testable without the heavy linkml package:

  • to_linkml() — GraphSpec → a LinkML schema dict (YAML-able).

  • load_linkml() — a LinkML schema (dict or YAML path) → GraphSpec.

  • generate_json_schema() — GraphSpec → JSON Schema (a $def per type, with enums/ranges/cardinality) for external validators.

  • generate_pydantic() — a Pydantic model per node/edge type for typed access.

Edges are LinkML classes flagged represents_relationship: true (the LinkML idiom for first-class, attribute-bearing relations).
